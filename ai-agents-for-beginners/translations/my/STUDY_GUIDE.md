# စတင်သူများအတွက် AI ကိုယ်စားလှယ်များ - သင်ကြားမှုလမ်းညွှန်နှင့် သင်တန်းအကျဥ်းချုပ်

ဤလမ်းညွှန်သည် "စတင်သူများအတွက် AI ကိုယ်စားလှယ်များ" သင်တန်းအကျဥ်းချုပ်ကို ပေးပြီး AI ကိုယ်စားလှယ်များ တည်ဆောက်ရာတွင် အဓိကသဘောတရားများ၊ ဖရိမ်ဝတ်များနှင့် ဒီဇိုင်းပုံစံများကို ရှင်းပြသည်။

## ၁။ AI ကိုယ်စားလှယ်များ၏ အ അരိုးအမြ story နည်းနိဒါန်း

**AI ကိုယ်စားလှယ်များသည် အဘယ်နည်း။**
AI ကိုယ်စားလှယ်များသည် ပိုမိုကြီးမားသော ဘာသာစကားမော်ဒယ်များ (LLMs) ၏ အင်အားများကို **ရည်ပုံတူကိရိယာများ**၊ **အသိပညာ** နှင့် **မှတ်ဉာဏ်** ချိတ်ဆက်ပေးခြင်းဖြင့် တိုးချဲ့ပေးသော စနစ်များဖြစ်သည်။ သင်ပေးထားသော ဒေတာအပေါ်မူတည်ပြီး စာသားထုတ်လုပ်ရာတွင်သာ အလားအလာရှိသော ရိုးရိုး LLM ချတ်ဘော့၏ ထက်၊ AI ကိုယ်စားလှယ်သည်:
- ပတ်ဝန်းကျင်ကို **ခံစား** တတ်သည် (အာရုံခံကိရိယာများသို့မဟုတ် အချက်အလက်ဝင်ရောက်မှုများအားဖြင့်)။
- ပြဿနာဖြေရှင်းပုံကို **ဆင်ခြေ** တတ်သည်။
- ပတ်ဝန်းကျင်ကို ပြောင်းလဲရန် **လှုပ်ရှား** တတ်သည် (လုပ်ဆောင်မှုကိရိယာများသို့မဟုတ် ကိရိယာများကို စစ်ဆင်ရေးရန်)။

**ကိုယ်စားလှယ်၏ အရေးပါတဲ့အစိတ်အပိုင်းများ:**
- **ပတ်ဝန်းကျင်**: ကိုယ်စားလှယ်၏ လည်ပတ်ရာနေရာ (ဥပမာ - ခရီးစာရင်းစနစ်)။
- **အာရုံခံကိရိယာများ**: သတင်းအချက်အလက်စုဆောင်းရန် မက်ခကိရိယာများ (ဥပမာ - API ဖတ်ခြင်း)။
- **လုပ်ဆောင်မှုကိရိယာများ**: လုပ်ဆောင်ချက်များ ပြုလုပ်ရန် မက်ခကိရိယာများ (ဥပမာ - အီးမေးလ်ပို့ခြင်း)။
- **ဥာဏ် (LLM)**: လုပ်ဆောင်ချက်များ ရွေးချယ်ခြင်းနှင့် စီမံခြင်းကိုလုပ်ဆောင်သည့် တာဝန်ရှိသူ။

## ၂။ ကိုယ်စားလှယ်ဖရိမ်းဝတ်များ

သင်တန်းသည် ကိုယ်စားလှယ်များ တည်ဆောက်ရာတွင် သုံးဦးတည်ရှိသောဖရိမ်းဝတ်များကို ဖော်ပြသည် -

| ဖရိမ်းဝတ် | အဓိကထားသောအချက် | သင့်လျော်သော အသုံးပြုမှု |
|-----------|------------------|--------------------------|
| **Semantic Kernel** | .NET/Python အတွက် ထုတ်လုပ်ပြင်ဆင်ပြီး SDK | လုပ်ငန်းအသုံးပြုမှုများ၊ ရှိပြီးသား ကုဒ်နှင့် AI ပေါင်းစပ်ခြင်း။ |
| **AutoGen** | အစုလိုက်အပြုံလိုက် ကိုယ်စားလှယ် ပူးပေါင်းဆောင်ရွက်မှု | အထူးပြု ကိုယ်စားလှယ်များ များများစွာ ဆက်သွယ်လျက်ရှိသော ရှုပ်ထွေးဖြစ်စိတ်အခြေအနေများ။ |
| **Azure AI Agent Service** | စီမံခန့်ခွဲထားသော မိမိနယ်ပယ်ဝန်ဆောင်မှု | built-in အခြေအနေစီမံခန့်ခွဲမှုနှင့် လုံခြုံနိုင်သော၊ နေရာချမ်းသာသော deployment။ |

## ၃။ ကိုယ်စားလှယ်ဒီဇိုင်းပုံစံများ

ဒီဇိုင်းပုံစံများသည် ကိုယ်စားလှယ်များ၏ လုပ်ဆောင်ပုံကို စတuktur်ဆက်ရန်ဖြစ်ပြီး ပြဿနာကို ယုံကြည်စိတ်ချစွာ ဖြေရှင်းရာတွင် အထောက်အကူပြုသည်။

### **ကိရိယာ အသုံးပြုမှု ပုံစံ** (သင်ခန်းစာ ၄)
ဤပုံစံသည် ကိုယ်စားလှယ်များအား ပြင်ပကမ္ဘာနှင့် ဆက်သွယ်နိုင်ရန် ခွင့်ပြုသည်။
- **ဆင်ခြေ**: ကိုယ်စားလှယ်အား "schema" (ရရှိနိုင်သောလုပ်ဆောင်မှုများနှင့် ၎င်းတို့၏ ပါရာမီတာစာရင်း) ပေးအပ်သည်။ LLM သည် အသုံးပြုသူ၏ တောင်းဆိုမှုအရ ဘယ်ကိရိယာကို ဘယ်အချက်အလက်ဖြင့် ခေါ်မည်ကို ဆုံးဖြတ်သည်။
- **စီးဆင်းမှု**: အသုံးပြုသူ တောင်းဆိုမှု -> LLM -> **ကိရိယာ ရွေးချယ်ခြင်း** -> **ကိရိယာ အကောင်အထည်ဖော်ခြင်း** -> LLM (ကိရိယာထုတ်လွှတ်ချက်နဲ့) -> နောက်ဆုံးဖြေကြားချက်။
- **အသုံးချမှုများ**: အချိန်နှင့်တပြေးညီ ဒေတာ (ရာသီဥတု၊ စတော့အိတ်ချိန်) ရယူခြင်း၊ တွက်ချက်ခြင်း၊ ကုဒ်အကောင်အထည်ဖော်ခြင်း။

### **စီမံပုံစံ** (သင်ခန်းစာ ၇)
ဤပုံစံသည် ကိုယ်စားလှယ်များအား ရှုပ်ထွေးပြီး အဆင့်မြင့်လုပ်ငန်းတစ်ခုစီမံရန်ခွင့်ပြုသည်။
- **ဆင်ခြေ**: ကိုယ်စားလှယ်သည် အစီအစဉ်အဆင့်မြင့်ရည်မှန်းချက်ကို အနည်းငယ်သော လိုအပ်ချက် အချို့ ပိုင်းခြားတုန့်ပြန်သည်။
- **လမ်းစဉ်များ**:
  - **တာဝန်ခွဲခြမ်းခြင်း**: "ခရီးစဉ် တစီစဉ်" ကို "လေယာဉ်ဘွတ်ကတ် ဝယ်ယူခြင်း", "ဟိုတယ် မှာခြင်း", "ကား ငှားခြင်း" ဆီ ခွဲခြမ်းခြင်း။
  - **အကြိမ်ကြိမ် စီမံခန့်ခွဲမှု**: ယခင်အဆင့်၏အထွက်စီမံချက်အပေါ်မူတည်ပြီး ပြန်လည်ဆန်းစစ်ခြင်း (ဥပမာ - လေယာဉ်လေယာဉ်နေရာ ပြည့်နေပါက မတူတဲ့ရက်ကို ရွေးချယ်ခြင်း)။
- **အကောင်အထည်ဖော်ခြင်း**: မကြာခဏ "စီမံခန့်ခွဲသူ" ကိုယ်စားလှယ် တစ်ဦးက စီမံအစီအစဉ်ကို (ဥပမာ JSON) ပြုပြင်ပေးပြီး၊ ထိုအစီအစဉ်ကို အခြားကိုယ်စားလှယ်များက တိုက်တွန်းပါသည်။

## ၄။ ဒီဇိုင်းအညွှန်းများ

ကိုယ်စားလှယ်များ ဒီဇိုင်းဆွဲရာတွင် သုံးအတိုင်းအတာကို စဉ်းစားပါ -
- **နေရာ**: ကိုယ်စားလှယ်များသည် လူများနှင့် အသိပညာကို ချိတ်ဆက်ရမည်၊ လွယ်ကူစွာ အသုံးပြုနိုင်သော်လည်း မထိခိုက်စေသင့်။
- **အချိန်**: ကိုယ်စားလှယ်များသည် *အတိတ်* မှ စာသင်ယူရမည်၊ *ယနေ့* တွင် သက်ဆိုင်သော အကြံပေးချက်များပေးရမည်၊ *အနာဂတ်* အတွက် လိုက်လျောညီထွေ ပြောင်းလဲနိုင်ရမည်။
- **အဓိက**: မသေချာမှုကိုလက်ခံပြီး သောက်သုံးသူက ထိန်းချုပ်နိုင်ရန်၊ ထင်ရှားပြတ်သားမှုဖြင့် ယုံကြည်မှု တည်ဆောက်ရမည်။

## ၅။ အဓိကသင်ခန်းစာအကျဥ်းချုပ်

- **သင်ခန်းစာ ၁**: ကိုယ်စားလှယ်များသည် စနစ်များ ဖြစ်ပြီး ကျွန်ုပ်တို့တင်မူတည်မော်ဒယ်များ မဟုတ်ပါ။ ၎င်းတို့သည် ခံစား၊ ဆင်ခြေ၊ လှုပ်ရှားသည်။
- **သင်ခန်းစာ ၂**: Semantic Kernel နှင့် AutoGen ကဲ့သို့ ဖရိမ်းဝတ်များသည် ကိရိယာခေါ်ဆိုမှုနှင့် အခြေအနေစီမံခန့်ခွဲမှု၏ရှုပ်ထွေးမှုကို ပယ်ချသည်။
- **သင်ခန်းစာ ၃**: ထင်ရှားပြတ်သားမှုနှင့် အသုံးပြုသူထိန်းချုပ်မှုကို စဉ်းစားပြီး ဒီဇိုင်းဆွဲပါ။
- **သင်ခန်းစာ ၄**: ကိရိယာများသည် ကိုယ်စားလှယ်၏ "လက်" ဖြစ်သည်။ LLM အနေဖြင့် ၎င်းတို့ကို အသုံးပြုနိုင်ရန် schema သတ်မှတ်ချက် မရှိမဖြစ်လိုအပ်သည်။
- **သင်ခန်းစာ ၇**: စီမံခန့်ခွဲမှုသည် ကိုယ်စားလှယ်၏ "အမှုဆောင်လုပ်ဆောင်ချက်" ဖြစ်ပြီး ရှုပ်ထွေးသော အလုပ်စဉ်များကို ရင်ဆိုင်ဖော်ဆောင်နိုင်စေသည်။
