# Штучні агенти для початківців - навчальний посібник та конспект курсу

Цей посібник надає конспект курсу "Штучні агенти для початківців" та пояснює ключові поняття, фреймворки та патерни проєктування для створення штучних агентів.

## 1. Вступ до Штучних агентів

**Що таке штучні агенти?**  
Штучні агенти — це системи, які розширюють можливості великих мовних моделей (LLMs), надаючи їм доступ до **інструментів**, **знань** та **пам’яті**. На відміну від звичайного чатбота на основі LLM, який лише генерує текст на основі тренувальних даних, штучний агент може:  
- **Сприймати** своє оточення (через сенсори або вхідні дані).  
- **Міркувати** про те, як розв’язати проблему.  
- **Діяти**, щоб змінити оточення (через виконавчі механізми або виконання інструментів).  

**Ключові компоненти агента:**  
- **Оточення**: Простір, де агент працює (наприклад, система бронювання).  
- **Сенсори**: Механізми збору інформації (наприклад, зчитування API).  
- **Виконавчі механізми**: Механізми для виконання дій (наприклад, надсилання електронних листів).  
- **Мозок (LLM)**: Двигун міркувань, який планує та приймає рішення, які дії виконати.  

## 2. Фреймворки агентів

У курсі розглядаються три основні фреймворки для створення агентів:

| Фреймворк | Фокус | Найкраще для |
|-----------|--------|-------------|
| **Semantic Kernel** | Готовий SDK для .NET/Python | Корпоративні застосунки, інтеграція ШІ з існуючим кодом. |  
| **AutoGen** | Співпраця багатьох агентів | Складні сценарії, що вимагають спеціалізованих агентів для взаємодії між собою. |  
| **Azure AI Agent Service** | Керована хмарна служба | Безпечне, масштабоване розгортання з вбудованим управлінням станом. |  

## 3. Патерни проєктування агентів

Патерни проєктування допомагають структурувати роботу агентів для надійного розв’язання задач.

### **Патерн Використання Інструментів** (Урок 4)  
Цей патерн дає змогу агентам взаємодіяти із зовнішнім світом.  
- **Концепція**: Агенту надається "схема" (список доступних функцій та їхніх параметрів). LLM вирішує, *який* інструмент викликати і з *якими* аргументами на основі запиту користувача.  
- **Потік**: Запит користувача -> LLM -> **Вибір інструменту** -> **Виконання інструменту** -> LLM (з результатом інструменту) -> Кінцева відповідь.  
- **Випадки використання**: Отримання актуальних даних (погода, ціни на акції), виконання розрахунків, запуск коду.  

### **Патерн Планування** (Урок 7)  
Цей патерн дає змогу агентам розв’язувати складні багатокрокові завдання.  
- **Концепція**: Агент розбиває загальну мету на послідовність менших підзадач.  
- **Підходи**:  
  - **Декомпозиція завдань**: Розбиття "Спланувати поїздку" на "Забронювати квиток", "Забронювати готель", "Орендувати авто".  
  - **Ітеративне планування**: Перегляд плану на основі результатів попередніх кроків (наприклад, якщо рейс заповнений, обрати іншу дату).  
- **Реалізація**: Зазвичай включає агента "Планувальник", який генерує структурований план (наприклад, у форматі JSON), що потім виконується іншими агентами.  

## 4. Принципи проєктування

При проєктуванні агентів враховуйте три виміри:  
- **Простір**: Агенти повинні поєднувати людей і знання, бути доступними, але непомітними.  
- **Час**: Агенти повинні навчатися на *минулому*, надавати релевантні підказки в *теперішньому* та адаптуватися до *майбутнього*.  
- **Сутність**: Приймайте невизначеність, але встановлюйте довіру через прозорість і контроль користувача.  

## 5. Підсумок ключових уроків

- **Урок 1**: Агенти — це системи, а не лише моделі. Вони сприймають, міркують і діють.  
- **Урок 2**: Фреймворки, як Semantic Kernel та AutoGen, абстрагують складність виклику інструментів та управління станом.  
- **Урок 3**: Проєктуйте з урахуванням прозорості та контролю користувача.  
- **Урок 4**: Інструменти — це "руки" агента. Визначення схеми має вирішальне значення для розуміння LLM, як їх використовувати.  
- **Урок 7**: Планування — це "виконавча функція" агента, що дає змогу вирішувати складні робочі процеси.
