# AI 代理程式初學者指南及課程摘要

本指南提供「AI 代理程式初學者」課程的摘要，並解釋建構 AI 代理程式的關鍵概念、框架及設計模式。

## 1. AI 代理程式介紹

**什麼是 AI 代理程式？**  
AI 代理程式是擴展大型語言模型（LLM）能力的系統，使其能存取**工具**、**知識**和**記憶**。與僅根據訓練資料生成文字的標準 LLM 聊天機器人不同，AI 代理程式可以：  
- **感知**其環境（透過感測器或輸入）。  
- **推理**如何解決問題。  
- **行動**以改變環境（透過執行器或工具執行）。  

**代理程式的關鍵組成部分：**  
- **環境**：代理程式運行的空間（例如訂票系統）。  
- **感測器**：收集資訊的機制（例如讀取 API）。  
- **執行器**：執行行動的機制（例如發送電子郵件）。  
- **大腦（LLM）**：規劃與決定採取何種行動的推理引擎。  

## 2. 代理框架

課程涵蓋三個主要建構代理程式的框架：

| 框架 | 重點 | 最適合應用 |
|-----------|-------|----------|
| **Semantic Kernel** | 用於 .NET/Python 的產業級 SDK | 企業應用，與現有代碼整合 AI。 |
| **AutoGen** | 多代理程式協作 | 需要多個專門代理程式互相交流的複雜場景。 |
| **Azure AI Agent Service** | 托管雲端服務 | 具備內建狀態管理的安全且可擴展部署。 |

## 3. 代理設計模式

設計模式幫助結構化代理的運作，從而可靠地解決問題。

### **工具使用模式**（第4課）  
此模式使代理程式能與外部世界互動。  
- **概念**：代理程式取得一個「架構」(schema)（可用函數及其參數清單）。LLM 根據使用者請求決定呼叫*哪個*工具和*用什麼*參數。  
- **流程**：使用者請求 -> LLM -> **工具選擇** -> **工具執行** -> LLM（帶工具輸出）-> 最終回答。  
- **應用場景**：查詢即時資料（天氣、股價）、執行計算、執行程式碼。  

### **規劃模式**（第7課）  
此模式使代理程式能解決複雜多步驟任務。  
- **概念**：代理程式將高階目標拆分為一連串較小子任務。  
- **方法**：  
  - **任務分解**：將「計劃旅行」拆解成「訂機票」、「訂旅店」、「租車」。  
  - **迭代規劃**：根據前一步輸出重新評估計劃（例如，若機票客滿，改選其他日期）。  
- **實作**：通常有一個「規劃者」代理產生結構化計劃（如 JSON），再由其他代理執行。  

## 4. 設計原則

設計代理程式時，應考量三個維度：  
- **空間**：代理應連結人與知識，易接近且不干擾。  
- **時間**：代理應從*過去*學習，在*當下*適時提醒，並為*未來*做調整。  
- **核心**：接受不確定性，但透過透明度與用戶控制建立信任。  

## 5. 主要課程重點摘要

- **第1課**：代理是系統，不只是模型。它們感知、推理、行動。  
- **第2課**：Semantic Kernel 和 AutoGen 這類框架抽象化工具呼叫與狀態管理的複雜性。  
- **第3課**：設計時應考慮透明度與用戶控制。  
- **第4課**：工具是代理的「手」。架構定義對 LLM 了解如何使用工具至關重要。  
- **第7課**：規劃是代理的「執行功能」，使其能處理複雜工作流程。
