# AI 에이전트 초보자용 - 학습 가이드 및 강의 요약

이 가이드는 "AI 에이전트 초보자용" 강의 내용을 요약하고 AI 에이전트 구축을 위한 핵심 개념, 프레임워크 및 설계 패턴을 설명합니다.

## 1. AI 에이전트 소개

**AI 에이전트란 무엇인가?**  
AI 에이전트는 도구, 지식, 메모리에 접근할 수 있도록 하여 대규모 언어 모델(LLM)의 능력을 확장하는 시스템입니다. 학습 데이터에 기반해 텍스트만 생성하는 일반 LLM 챗봇과 달리, AI 에이전트는:  
- 환경을 **인식**합니다(센서 또는 입력을 통해).  
- 문제 해결 방법에 대해 **추론**합니다.  
- 환경을 변화시키기 위해 **행동**합니다(액추에이터 또는 도구 실행을 통해).  

**에이전트의 주요 구성 요소:**  
- **환경**: 에이전트가 작동하는 공간(예: 예약 시스템).  
- **센서**: 정보를 수집하는 메커니즘(예: API 읽기).  
- **액추에이터**: 행동을 수행하는 메커니즘(예: 이메일 전송).  
- **두뇌(LLM)**: 계획하고 어떤 행동을 취할지 결정하는 추론 엔진.  

## 2. 에이전트 프레임워크

이 강의는 에이전트 구축을 위한 세 가지 주요 프레임워크를 다룹니다:

| 프레임워크 | 중점 | 적합 대상 |
|-----------|-------|----------|
| **Semantic Kernel** | .NET/Python용 프로덕션 준비 SDK | 엔터프라이즈 애플리케이션, 기존 코드와 AI 통합. |
| **AutoGen** | 다중 에이전트 협업 | 여러 전문 에이전트가 상호 소통하는 복잡한 시나리오. |
| **Azure AI Agent Service** | 관리형 클라우드 서비스 | 내장 상태 관리로 보안 및 확장 가능 배포. |

## 3. 에이전트 설계 패턴

설계 패턴은 에이전트가 문제를 신뢰성 있게 해결하도록 작동 방식을 구조화하는 데 도움을 줍니다.

### **도구 사용 패턴** (4과)  
이 패턴은 에이전트가 외부 세계와 상호작용할 수 있게 합니다.  
- **개념**: 에이전트는 사용 가능한 함수와 매개변수 목록인 "스키마"를 제공합니다. LLM은 사용자 요청에 기반해 *어떤* 도구를 호출할지, 그리고 *어떤* 인수로 호출할지 결정합니다.  
- **흐름**: 사용자 요청 -> LLM -> **도구 선택** -> **도구 실행** -> 도구 출력과 함께 LLM -> 최종 응답.  
- **사용 사례**: 실시간 데이터 조회(날씨, 주가), 계산 수행, 코드 실행.  

### **계획 패턴** (7과)  
이 패턴은 에이전트가 복잡하고 다단계 작업을 해결할 수 있게 합니다.  
- **개념**: 에이전트는 상위 목표를 더 작은 하위 작업 시퀀스로 분해합니다.  
- **접근법**:  
  - **작업 분해**: "여행 계획"을 "항공편 예약", "호텔 예약", "차 렌트"로 나누기.  
  - **반복 계획**: 이전 단계 결과를 기반으로 계획 재평가(예: 항공편이 만석이면 다른 날짜 선택).  
- **구현**: 일반적으로 구조화된 계획(예: JSON)을 생성하는 "플래너" 에이전트가 있고, 이를 다른 에이전트가 실행합니다.  

## 4. 설계 원칙

에이전트 설계 시 세 가지 차원을 고려하십시오:  
- **공간(Space)**: 에이전트는 사람과 지식을 연결하며, 접근 가능하면서도 방해되지 않아야 합니다.  
- **시간(Time)**: 에이전트는 *과거*로부터 학습하고, *현재* 적절한 알림을 제공하며, *미래*에 적응해야 합니다.  
- **핵심(Core)**: 불확실성을 수용하되, 투명성과 사용자 제어를 통해 신뢰를 구축해야 합니다.  

## 5. 주요 강의 요약

- **1과**: 에이전트는 단순 모델이 아니라 시스템입니다. 인식하고, 추론하며, 행동합니다.  
- **2과**: Semantic Kernel, AutoGen과 같은 프레임워크는 도구 호출과 상태 관리를 추상화합니다.  
- **3과**: 투명성과 사용자 제어를 고려해 설계하십시오.  
- **4과**: 도구는 에이전트의 "손"입니다. 스키마 정의는 LLM이 도구 사용법을 이해하는 데 중요합니다.  
- **7과**: 계획은 에이전트의 "집행 기능"으로 복잡한 작업 흐름을 다루게 합니다.
