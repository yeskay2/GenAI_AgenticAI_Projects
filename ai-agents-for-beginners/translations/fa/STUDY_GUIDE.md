# مأموران هوش مصنوعی برای مبتدیان - راهنمای مطالعه و خلاصه دوره

این راهنما خلاصه‌ای از دوره «مأموران هوش مصنوعی برای مبتدیان» را ارائه می‌دهد و مفاهیم کلیدی، چارچوب‌ها و الگوهای طراحی برای ساخت مأموران هوش مصنوعی را توضیح می‌دهد.

## ۱. مقدمه‌ای بر مأموران هوش مصنوعی

**مأموران هوش مصنوعی چیستند؟**  
مأموران هوش مصنوعی سیستم‌هایی هستند که قابلیت‌های مدل‌های زبان بزرگ (LLMها) را با دادن دسترسی به **ابزارها**، **دانش** و **حافظه** گسترش می‌دهند. برخلاف یک چت‌بات استاندارد LLM که تنها متن را بر اساس داده‌های آموزشی تولید می‌کند، یک مأمور هوش مصنوعی می‌تواند:  
- **محیط خود را درک کند** (از طریق حسگرها یا ورودی‌ها).  
- **استدلال کند** درباره نحوه حل یک مسئله.  
- **عمل کند** تا محیط را تغییر دهد (از طریق عملگرها یا اجرای ابزار).

**اجزای کلیدی یک مأمور:**  
- **محیط**: فضایی که مأمور در آن عمل می‌کند (مثلاً یک سیستم رزرو).  
- **حسگرها**: مکانیزم‌هایی برای جمع‌آوری اطلاعات (مثلاً خواندن یک API).  
- **عملگرها**: مکانیزم‌هایی برای انجام اقدامات (مثلاً ارسال ایمیل).  
- **مغز (LLM)**: موتور استدلال که برنامه‌ریزی می‌کند و تصمیم می‌گیرد چه اقداماتی انجام شود.

## ۲. چارچوب‌های مأمورانه

دوره سه چارچوب اصلی برای ساخت مأموران را پوشش می‌دهد:

| چارچوب | تمرکز | بهترین کاربرد |
|---------|-------|--------------|
| **Semantic Kernel** | کیت توسعه آماده تولید برای .NET/Python | برنامه‌های سازمانی، ادغام هوش مصنوعی با کدهای موجود. |  
| **AutoGen** | همکاری میان چند مأمور | سناریوهای پیچیده که نیازمند چند مأمور متخصص هستند که با هم گفتگو می‌کنند. |  
| **Azure AI Agent Service** | سرویس ابری مدیریت شده | استقرار امن و مقیاس‌پذیر با مدیریت وضعیت داخلی. |

## ۳. الگوهای طراحی مأمورانه

الگوهای طراحی به ساختاردهی نحوه عملکرد مأموران برای حل مشکلات به طور مطمئن کمک می‌کنند.

### **الگوی استفاده از ابزار** (درس ۴)  
این الگو امکان تعامل مأموران با دنیای بیرون را فراهم می‌کند.  
- **مفهوم**: به مأمور یک «طرح‌واره» داده می‌شود (لیستی از توابع موجود و پارامترهای آن‌ها). LLM تصمیم می‌گیرد *کدام* ابزار را با *چه* آرگومان‌هایی براساس درخواست کاربر فراخوانی کند.  
- **جریان**: درخواست کاربر -> LLM -> **انتخاب ابزار** -> **اجرای ابزار** -> LLM (با خروجی ابزار) -> پاسخ نهایی.  
- **موارد استفاده**: بازیابی داده‌های بلادرنگ (آب‌وهوا، قیمت سهام)، انجام محاسبات، اجرای کد.

### **الگوی برنامه‌ریزی** (درس ۷)  
این الگو به مأموران امکان حل وظایف پیچیده و چندمرحله‌ای را می‌دهد.  
- **مفهوم**: مأمور هدف سطح بالا را به دنباله‌ای از وظایف کوچک‌تر خرد می‌کند.  
- **رویکردها**:  
  - **کاهش وظایف**: شکستن «برنامه‌ریزی سفر» به «رزرو پرواز»، «رزرو هتل»، «اجاره خودرو».  
  - **برنامه‌ریزی تکراری**: بازنگری برنامه بر اساس خروجی مراحل قبلی (مثلاً اگر پرواز پر باشد، تاریخ دیگری انتخاب شود).  
- **پیاده‌سازی**: معمولاً شامل یک مأمور «برنامه‌ریز» است که برنامه‌ای ساختاریافته (مثلاً JSON) تولید می‌کند که سپس توسط مأموران دیگر اجرا می‌شود.

## ۴. اصول طراحی

هنگام طراحی مأموران، سه بُعد را در نظر بگیرید:  
- **فضا**: مأموران باید مردم و دانش را به هم متصل کنند، قابل دسترس اما کم‌مزاحمت باشند.  
- **زمان**: مأموران باید از *گذشته* بیاموزند، در *حال* توصیه‌های مرتبط ارائه دهند و برای *آینده* سازگار شوند.  
- **هسته**: ابهام را بپذیرید اما از طریق شفافیت و کنترل کاربر اعتماد ایجاد کنید.

## ۵. خلاصه درس‌های کلیدی

- **درس ۱**: مأموران سیستم هستند، نه فقط مدل. آن‌ها درک می‌کنند، استدلال می‌کنند و عمل می‌کنند.  
- **درس ۲**: چارچوب‌هایی مانند Semantic Kernel و AutoGen پیچیدگی فراخوانی ابزار و مدیریت وضعیت را خلاصه می‌کنند.  
- **درس ۳**: طراحی را با شفافیت و کنترل کاربر مد نظر قرار دهید.  
- **درس ۴**: ابزارها «دست‌های» مأمور هستند. تعریف طرح‌واره برای فهمیدن نحوه استفاده آن‌ها توسط LLM حیاتی است.  
- **درس ۷**: برنامه‌ریزی «وظیفه اجرایی» مأمور است که به آن امکان می‌دهد جریان‌های کاری پیچیده را مدیریت کند.
