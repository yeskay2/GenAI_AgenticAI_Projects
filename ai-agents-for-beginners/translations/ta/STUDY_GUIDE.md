# தொடக்கத்திற்கான ஏআই முகவர்கள் - படிப்பு வழிகாட்டு மற்றும் பாடநெறி சுருக்கம்

இந்த வழிகாட்டு "தொடக்கத்திற்கான ஏআই முகவர்கள்" என்ற பாடநெறியின் சுருக்கத்தையும், ஏআই முகவர்களை உருவாக்குவதற்கான முக்கியக் கருத்துக்கள், கட்டமைப்புகள் மற்றும் வடிவமைப்பு முறைகளை விளக்குகிறது.

## 1. ஏഐ முகவர்கள் அறிமுகம்

**ஏன் ஏஐ முகவர்கள்?**  
ஏஐ முகவர்கள் பெரிய மொழி மாதிரிகளின் (LLMs) திறன்களை **கருவிகள்**, **அறிவு** மற்றும் **நினைவகம்** மூலம் விரிவுபடுத்தும் அமைப்புகள். பயிற்சி தரவுக்கு அடிப்படையாய்த் உரை உருவாக்கும் சாதாரண LLM உரையாடி போல அல்லாமல், ஒரு ஏஐ முகவர்க்:  
- **சூழலை உணர்கிறது** (அறிவிப்பாளர்கள் அல்லது உள்ளீடுகளின் மூலம்).  
- **தீர்க்கத் தகுதி** கொண்டது பிரச்சினையை எவ்வாறு தீர்க்க வேண்டும் என்பதை ஆராய்கிறது.  
- **செயல் படுத்துகிறது** சூழலை மாற்ற (அஞ்சக் கருவிகள் அல்லது கருவி செயலாக்கங்கள் மூலம்).

**முகவரின் முக்கிய கூறுகள்:**  
- **சூழல்**: முகவர் இயங்கும் இடம் (எ.கா., பரிசீலனை அமைப்பு).  
- **அறிவிப்பாளர்கள்**: தகவல் சேகரிக்கும் இயந்திரங்கள் (எ.கா., API வாசிப்பு).  
- **செயலாக்கிகள்**: செயல்கள் செய்யும் இயந்திரங்கள் (எ.கா., மின்னஞ்சல் அனுப்புதல்).  
- **மூளை (LLM)**: எந்த செயல்களை எடுக்க வேண்டும் என்று திட்டமிடும் மற்றும் தீர்மானிக்கும் காரண engines.

## 2. முகவர் கட்டமைப்புகள்

முகவர்களை உருவாக்குவதற்கான மூன்று முக்கிய கட்டமைப்புகள் பாடநெறியில் உள்ளன:

| கட்டமைப்பு | கவனம் | சிறந்த பயன்பாடு |
|-----------|-------|------------------|
| **Semantic Kernel** | .NET/Python க்கான தயாரிப்பு-தயாரான SDK | நிறுவன பயன்பாடுகள், ஏஐயை உள்ளமைந்த குறியீட்டுடன் இணைத்தல். |
| **AutoGen** | பல முகவர்கள் ஒருங்கிணைவு | பல நிபுண முகவர்கள் பேசும் சிக்கலான சூழல்கள். |
| **Azure AI Agent Service** | மேய்யப்பட்ட மெகுகிய சேவை | பாதுகாப்பான, விரிவாக்கக்கூடிய டிபிளாய்மென்ட் மற்றும் நிலை நிர்வாகம். |

## 3. முகவர் வடிவமைப்பு முறைகள்

வடிவமைப்பு முறைகள் முகவர்களுக்கான நடைமுறைகளை கட்டமைக்க உதவுகின்றன.

### **கருவி பயன்பாடு முறை** (பாடம் 4)  
இந்த முறை முகவர்களுக்கு வெளிப்புற உலகுடன் தொடர்பு கொள்ள உதவுகிறது.  
- **கருத்து**: முகவருக்கு "வட்டிகை" (வழங்கப்பட்ட செயல்பாடுகளின் பட்டியல் மற்றும் அவற்றின் அளவுருக்கள்) கொடுக்கப்படுகிறது. LLM பயனர் கோரிக்கையின் அடிப்படையில் *என்ன* கருவியை எடுக்க வேண்டும் என்பதையும், *எந்த* அளவுருக்களுடன் அழைக்க வேண்டும் என்பதையும் தீர்மானிக்கிறது.  
- **ஓட்டம்**: பயனர் கோரிக்கை -> LLM -> **கருவி தேர்வு** -> **கருவி செயலாக்கம்** -> LLM (கருவி வெளியீட்டுடன்) -> இறுதி பதில்.  
- **பயன்பாடுகள்**: நேரடி தரவுகளை (வானிலை, பங்கு விலை), கணக்கீடுகளை, குறியீடு செயலாக்கத்தைக் கொண்டுவருதல்.

### **திட்டமிடல் முறை** (பாடம் 7)  
இந்த முறை மிகுந்த அடுக்கு பணிகளை தீர்க்க உதவுகிறது.  
- **கருத்து**: இரண்டு உருண்ட குறிக்கோளை சின்ன துணை பணிகளாக பிரிக்கும்.  
- **நடவடிக்கைகள்**:  
  - **பணி பிரிப்பு**: "பயணம் திட்டமிடு" என்பதனை "பயணியுழை பதிவு செய்", "ஹோட்டல் பதிவு செய்", "கார் வாடகை" என்பதாகப் பிரிக்க.  
  - **மறுசெயலாக்க திட்டமிடல்**: முன் படிகளின் முடிவுகளின் அடிப்படையில் திட்டத்தை மறுபார்வை செய் (பயணியுழை முழுமையானால் வேறு தேதியை தேர்வு செய்).  
- **உள்ளமைப்பு**: பொதுவாக "திட்டமிடல் முகவர்" ஒன்று கட்டமைப்பான திட்டத்தை (எ.கா., JSON) உருவாக்கி பிற முகவர்கள் அதை செயலாக்குகின்றனர்.

## 4. வடிவமைப்பு கோட்பாடுகள்

முகவர்களை வடிவமைக்கும்போது மூன்று பரிமாணங்களை கருத்தில் கொண்டு செய்ய வேண்டும்:  
- **இடம்**: முகவர்கள் மனிதர்கள் மற்றும் அறிவை இணைத்து, அணுகக்கூடிய ஆனால் கவனத்தைத் தடுக்கும் வகையில் இருக்க வேண்டும்.  
- **நேரம்**: கடந்த காலத்திலிருந்து கற்றுக்கொள், இப்போது தொடர்புடைய உதவிகளை வழங்கு, மற்றும் எதிர்காலத்திற்குப் பொருந்தும் வகையில் சரிசெய்.  
- **பொது**: சந்தேகத்தையே ஏற்று, ஆனால் வெளிப்பாடுத்தன்மை மற்றும் பயனர் கட்டுப்பாட்டின் மூலம் நம்பிக்கையை உருவாக்கு.

## 5. முக்கிய பாடங்களின் சுருக்கம்

- **பாடம் 1**: முகவர்கள் என்பது மாதிரிகள் அல்ல. அவை உணர்கின்றன, தீர்க்கின்றன மற்றும் செயல் படுத்துகின்றன.  
- **பாடம் 2**: Semantic Kernel மற்றும் AutoGen போன்ற கட்டமைப்புகள் கருவி அழைப்பு மற்றும் நிலை நிர்வாகத்தின் சிக்கலை சுருக்குகின்றன.  
- **பாடம் 3**: வெளிப்பாடும் பயனர் கட்டுப்பாடும் முக்கியமாக உருவாக்குக.  
- **பாடம் 4**: கருவிகள் முகவரின் "கைகள்". Schema வரையறு என்பது LLM க்கு அவற்றைப் பயன்படுத்துவது எப்படி என்பதை புரிய உதவுகிறது.  
- **பாடம் 7**: திட்டமிடல் என்பது முகவரின் "நிர்வாக செயல்பாடு", சிக்கலான பணிகள் செய்ய உதவுகிறது.
