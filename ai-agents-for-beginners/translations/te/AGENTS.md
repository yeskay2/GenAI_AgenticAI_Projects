# AGENTS.md

## ప్రాజెక్ట్ అవలోకనం

ఈ రిపోజిటరీ "AI ఏజెంట్స్ ఫర్ బిగినర్స్" అనే సమగ్ర విద్యా కోర్సును కలిగి ఉంది, ఇది AI ఏజెంట్స్ నిర్మాణానికి అవసరమైన అన్ని విషయాలను నేర్పుతుంది. ఈ కోర్సులో 15+ పాఠాలు ఉన్నాయి, ఇవి ప్రాథమికాలు, డిజైన్ ప్యాటర్న్స్, ఫ్రేమ్‌వర్క్స్, మరియు AI ఏజెంట్స్ ప్రొడక్షన్ డిప్లాయ్‌మెంట్‌ను కవర్ చేస్తాయి.

**ప్రధాన సాంకేతికతలు:**
- Python 3.12+
- ఇంటరాక్టివ్ లెర్నింగ్ కోసం Jupyter నోట్‌బుక్స్
- AI ఫ్రేమ్‌వర్క్స్: సెమాంటిక్ కర్నెల్, ఆటోజెన్, మైక్రోసాఫ్ట్ ఏజెంట్ ఫ్రేమ్‌వర్క్ (MAF)
- Azure AI సర్వీసెస్: Azure AI Foundry, Azure AI Agent Service
- GitHub మోడల్స్ మార్కెట్‌ప్లేస్ (ఉచిత టియర్ అందుబాటులో ఉంది)

**ఆర్కిటెక్చర్:**
- పాఠాల ఆధారిత నిర్మాణం (00-15+ డైరెక్టరీలు)
- ప్రతి పాఠం కలిగి ఉంటుంది: README డాక్యుమెంటేషన్, కోడ్ నమూనాలు (Jupyter నోట్‌బుక్స్), మరియు చిత్రాలు
- ఆటోమేటెడ్ అనువాద వ్యవస్థ ద్వారా బహుభాషా మద్దతు
- ప్రతి పాఠానికి బహుళ ఫ్రేమ్‌వర్క్ ఎంపికలు (సెమాంటిక్ కర్నెల్, ఆటోజెన్, Azure AI Agent Service)

## సెటప్ కమాండ్లు

### అవసరమైనవి
- Python 3.12 లేదా అంతకంటే ఎక్కువ
- GitHub ఖాతా (GitHub మోడల్స్ కోసం - ఉచిత టియర్)
- Azure సబ్‌స్క్రిప్షన్ (ఐచ్ఛికం, Azure AI సర్వీసెస్ కోసం)

### ప్రారంభ సెటప్

1. **రిపోజిటరీని క్లోన్ చేయండి లేదా ఫోర్క్ చేయండి:**
   ```bash
   gh repo fork microsoft/ai-agents-for-beginners --clone
   # లేదా
   git clone https://github.com/microsoft/ai-agents-for-beginners.git
   cd ai-agents-for-beginners
   ```

2. **Python వర్చువల్ ఎన్విరాన్‌మెంట్ క్రియేట్ చేసి యాక్టివేట్ చేయండి:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # విండోస్‌లో: venv\Scripts\activate
   ```

3. **డిపెండెన్సీలను ఇన్‌స్టాల్ చేయండి:**
   ```bash
   pip install -r requirements.txt
   ```

4. **ఎన్విరాన్‌మెంట్ వేరియబుల్స్ సెట్ చేయండి:**
   ```bash
   cp .env.example .env
   # మీ API కీలు మరియు ఎండ్‌పాయింట్లతో .env ను సవరించండి
   ```


### అవసరమైన ఎన్విరాన్‌మెంట్ వేరియబుల్స్

**GitHub మోడల్స్ (ఉచిత):**
- `GITHUB_TOKEN` - GitHub నుండి వ్యక్తిగత యాక్సెస్ టోకెన్

**Azure AI సర్వీసెస్** (ఐచ్ఛికం):
- `PROJECT_ENDPOINT` - Azure AI Foundry ప్రాజెక్ట్ ఎండ్‌పాయింట్
- `AZURE_OPENAI_API_KEY` - Azure OpenAI API కీ
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI ఎండ్‌పాయింట్ URL
- `AZURE_OPENAI_CHAT_DEPLOYMENT_NAME` - చాట్ మోడల్ కోసం డిప్లాయ్‌మెంట్ పేరు
- `AZURE_OPENAI_EMBEDDING_DEPLOYMENT_NAME` - ఎంబెడ్డింగ్స్ కోసం డిప్లాయ్‌మెంట్ పేరు
- `.env.example` లో చూపిన అదనపు Azure కాన్ఫిగరేషన్

## డెవలప్‌మెంట్ వర్క్‌ఫ్లో

### Jupyter నోట్‌బుక్స్ నడపడం

ప్రతి పాఠం వివిధ ఫ్రేమ్‌వర్క్స్ కోసం బహుళ Jupyter నోట్‌బుక్స్ కలిగి ఉంటుంది:

1. **Jupyter ప్రారంభించండి:**
   ```bash
   jupyter notebook
   ```

2. **ఒక పాఠం డైరెక్టరీకి వెళ్లండి** (ఉదా., `01-intro-to-ai-agents/code_samples/`)

3. **నోట్‌బుక్స్ ఓపెన్ చేసి నడపండి:**
   - `*-semantic-kernel.ipynb` - సెమాంటిక్ కర్నెల్ ఫ్రేమ్‌వర్క్ ఉపయోగించి
   - `*-autogen.ipynb` - ఆటోజెన్ ఫ్రేమ్‌వర్క్ ఉపయోగించి
   - `*-python-agent-framework.ipynb` - మైక్రోసాఫ్ట్ ఏజెంట్ ఫ్రేమ్‌వర్క్ (Python) ఉపయోగించి
   - `*-dotnet-agent-framework.ipynb` - మైక్రోసాఫ్ట్ ఏజెంట్ ఫ్రేమ్‌వర్క్ (.NET) ఉపయోగించి
   - `*-azureaiagent.ipynb` - Azure AI Agent Service ఉపయోగించి

### వివిధ ఫ్రేమ్‌వర్క్స్‌తో పని చేయడం

**సెమాంటిక్ కర్నెల్ + GitHub మోడల్స్:**
- GitHub ఖాతాతో ఉచిత టియర్ అందుబాటులో ఉంది
- నేర్చుకోవడం మరియు ప్రయోగాల కోసం అనుకూలం
- ఫైల్ నమూనా: `*-semantic-kernel*.ipynb`

**ఆటోజెన్ + GitHub మోడల్స్:**
- GitHub ఖాతాతో ఉచిత టియర్ అందుబాటులో ఉంది
- బహుళ ఏజెంట్ ఆర్కెస్ట్రేషన్ సామర్థ్యాలు
- ఫైల్ నమూనా: `*-autogen.ipynb`

**మైక్రోసాఫ్ట్ ఏజెంట్ ఫ్రేమ్‌వర్క్ (MAF):**
- మైక్రోసాఫ్ట్ నుండి తాజా ఫ్రేమ్‌వర్క్
- Python మరియు .NET లో అందుబాటులో ఉంది
- ఫైల్ నమూనా: `*-agent-framework.ipynb`

**Azure AI Agent Service:**
- Azure సబ్‌స్క్రిప్షన్ అవసరం
- ప్రొడక్షన్-రెడీ ఫీచర్లు
- ఫైల్ నమూనా: `*-azureaiagent.ipynb`

## టెస్టింగ్ సూచనలు

ఇది విద్యా రిపోజిటరీ, ఉదాహరణ కోడ్‌తో, ఆటోమేటెడ్ టెస్టులు లేని ప్రొడక్షన్ కోడ్ కాదు. మీ సెటప్ మరియు మార్పులను ధృవీకరించడానికి:

### మాన్యువల్ టెస్టింగ్

1. **Python ఎన్విరాన్‌మెంట్‌ను టెస్ట్ చేయండి:**
   ```bash
   python --version  # 3.12+ ఉండాలి
   pip list | grep -E "(autogen|semantic-kernel|azure-ai)"
   ```

2. **నోట్‌బుక్ ఎగ్జిక్యూషన్‌ను టెస్ట్ చేయండి:**
   ```bash
   # నోట్‌బుక్‌ను స్క్రిప్ట్‌గా మార్చి అమలు చేయండి (ఇంపోర్ట్‌లను పరీక్షిస్తుంది)
   jupyter nbconvert --to script <lesson-folder>/code_samples/<notebook>.ipynb --stdout | python
   ```

3. **ఎన్విరాన్‌మెంట్ వేరియబుల్స్‌ను ధృవీకరించండి:**
   ```bash
   python -c "import os; from dotenv import load_dotenv; load_dotenv(); print('✓ GITHUB_TOKEN' if os.getenv('GITHUB_TOKEN') else '✗ GITHUB_TOKEN missing')"
   ```


### వ్యక్తిగత నోట్‌బుక్స్ నడపడం

Jupyter లో నోట్‌బుక్స్ ఓపెన్ చేసి, క్రమంగా సెల్స్ ఎగ్జిక్యూట్ చేయండి. ప్రతి నోట్‌బుక్ స్వతంత్రంగా ఉంటుంది మరియు ఇందులో ఉంటుంది:
- ఇంపోర్ట్ స్టేట్‌మెంట్స్
- కాన్ఫిగరేషన్ లోడింగ్
- ఉదాహరణ ఏజెంట్ అమలు
- మార్క్‌డౌన్ సెల్స్‌లో ఊహించిన అవుట్‌పుట్‌లు

   - ప్రభావిత నోట్‌బుక్స్‌ను పూర్తిగా నడపండి
   - అన్ని సెల్స్ ఎర్రర్స్ లేకుండా ఎగ్జిక్యూట్ అవుతున్నాయో చూడండి
   - అవుట్‌పుట్‌లు సరైనవిగా ఉన్నాయో ధృవీకరించండి

   - కొత్త కాన్సెప్ట్‌లను చేర్చినప్పుడు README.md నవీకరించండి
   - క్లిష్టమైన కోడ్ కోసం నోట్‌బుక్స్‌లో కామెంట్స్ చేర్చండి
   - మార్క్‌డౌన్ సెల్స్ ఉద్దేశాన్ని వివరించాలి

   - `.env` ఫైళ్లను కమిట్ చేయవద్దు (`.env.example` ఉపయోగించండి)
   - `venv/` లేదా `__pycache__/` డైరెక్టరీలను కమిట్ చేయవద్దు
   - కాన్సెప్ట్‌లను చూపించడానికి అవసరమైనప్పుడు నోట్‌బుక్ అవుట్‌పుట్‌లను ఉంచండి

## అదనపు గమనికలు

### సాధారణ సమస్యలు

1. **Python వెర్షన్ తేడా:**
   - Python 3.12+ ఉపయోగించబడిందని నిర్ధారించుకోండి
   - కొన్ని ప్యాకేజీలు పాత వెర్షన్లతో పని చేయకపోవచ్చు
   - Python వెర్షన్‌ను స్పష్టంగా పేర్కొనడానికి `python3 -m venv` ఉపయోగించండి

2. **ఎన్విరాన్‌మెంట్ వేరియబుల్స్:**
   - ఎల్లప్పుడూ `.env.example` నుండి `.env` క్రియేట్ చేయండి
   - `.env` ఫైల్‌ను కమిట్ చేయవద్దు (ఇది `.gitignore` లో ఉంది)
   - GitHub టోకెన్ సరైన అనుమతులు కలిగి ఉండాలి

3. **ప్యాకేజీ విరుద్ధతలు:**
   - కొత్త వర్చువల్ ఎన్విరాన్‌మెంట్ ఉపయోగించండి
   - వ్యక్తిగత ప్యాకేజీలకు బదులుగా `requirements.txt` నుండి ఇన్‌స్టాల్ చేయండి
   - కొన్ని నోట్‌బుక్స్‌లో వారి మార్క్‌డౌన్ సెల్స్‌లో పేర్కొన్న అదనపు ప్యాకేజీలు అవసరం

4. **Azure సర్వీసెస్:**
   - Azure AI సర్వీసెస్‌కు చురుకైన సబ్‌స్క్రిప్షన్ అవసరం
   - కొన్ని ఫీచర్లు ప్రాంతానికి ప్రత్యేకంగా ఉంటాయి
   - GitHub మోడల్స్‌కు ఉచిత టియర్ పరిమితులు వర్తిస్తాయి
