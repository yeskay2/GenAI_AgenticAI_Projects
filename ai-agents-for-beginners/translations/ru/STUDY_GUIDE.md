# AI-агенты для начинающих — Учебное руководство и краткое содержание курса

Это руководство предоставляет краткое содержание курса «AI-агенты для начинающих» и объясняет ключевые концепции, фреймворки и шаблоны проектирования для создания AI-агентов.

## 1. Введение в AI-агентов

**Что такое AI-агенты?**  
AI-агенты — это системы, которые расширяют возможности больших языковых моделей (LLM), предоставляя им доступ к **инструментам**, **знаниям** и **памяти**. В отличие от стандартного чатбота на базе LLM, который только генерирует текст на основе обучающих данных, AI-агент может:  
- **Воспринимать** окружающую среду (через сенсоры или входные данные).  
- **Рассуждать** о том, как решить задачу.  
- **Действовать**, чтобы изменить окружающую среду (через актуаторы или выполнение инструментов).

**Ключевые компоненты агента:**  
- **Окружающая среда**: пространство, в котором действует агент (например, система бронирования).  
- **Сенсоры**: механизмы для сбора информации (например, чтение API).  
- **Актуаторы**: механизмы для выполнения действий (например, отправка email).  
- **Мозг (LLM)**: движок рассуждений, который планирует и выбирает действия.

## 2. Фреймворки для агентов

В курсе рассматриваются три основных фреймворка для создания агентов:

| Фреймворк | Фокус | Лучшее применение |
|-----------|-------|-------------------|
| **Semantic Kernel** | Готовый к производству SDK для .NET/Python | Корпоративные приложения, интеграция ИИ с существующим кодом. |
| **AutoGen** | Мультиагентное сотрудничество | Сложные сценарии, требующие взаимодействия нескольких специализированных агентов. |
| **Azure AI Agent Service** | Управляемый облачный сервис | Безопасное и масштабируемое развёртывание с встроенным управлением состоянием. |

## 3. Шаблоны проектирования агентов

Шаблоны проектирования помогают структурировать работу агентов для надёжного решения задач.

### **Шаблон использования инструментов** (Урок 4)  
Этот шаблон позволяет агентам взаимодействовать с внешним миром.  
- **Концепция**: агенту предоставляется «схема» (список доступных функций и их параметров). LLM решает, *какой* инструмент вызвать и с *какими* аргументами на основе запроса пользователя.  
- **Последовательность**: Запрос пользователя -> LLM -> **Выбор инструмента** -> **Выполнение инструмента** -> LLM (с выводом инструмента) -> Итоговый ответ.  
- **Случаи использования**: Получение данных в реальном времени (погода, цены акций), выполнение вычислений, запуск кода.

### **Шаблон планирования** (Урок 7)  
Этот шаблон позволяет агентам решать сложные задачи с несколькими шагами.  
- **Концепция**: агент разбивает высокоуровневую цель на ряд меньших подзадач.  
- **Подходы**:  
  - **Декомпозиция задач**: разделение «Спланировать поездку» на «Забронировать билет», «Забронировать отель», «Взять машину напрокат».  
  - **Итеративное планирование**: переоценка плана на основе результатов предыдущих шагов (например, если билет на самолет продан, выбрать другую дату).  
- **Реализация**: часто включает агента «Планировщика», который генерирует структурированный план (например, в формате JSON), затем выполняемый другими агентами.

## 4. Принципы проектирования

При проектировании агентов учитывайте три измерения:  
- **Пространство**: агенты должны соединять людей и знания, быть доступными, но ненавязчивыми.  
- **Время**: агенты должны учиться на *прошлом*, предоставлять релевантные подсказки в *настоящем* и адаптироваться к *будущему*.  
- **Ядро**: принимайте неопределённость, но создавайте доверие через прозрачность и контроль пользователя.

## 5. Краткое содержание ключевых уроков

- **Урок 1**: Агенты — это системы, а не просто модели. Они воспринимают, рассуждают и действуют.  
- **Урок 2**: Фреймворки, такие как Semantic Kernel и AutoGen, абстрагируют сложность вызова инструментов и управления состоянием.  
- **Урок 3**: Проектируйте с учётом прозрачности и контроля со стороны пользователя.  
- **Урок 4**: Инструменты — это «руки» агента. Определение схемы критически важно для понимания LLM, как ими пользоваться.  
- **Урок 7**: Планирование — это «исполнительная функция» агента, позволяющая справляться с комплексными рабочими процессами.
