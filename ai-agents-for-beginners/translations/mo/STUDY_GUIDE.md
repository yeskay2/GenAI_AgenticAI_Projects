# AI Agents for Beginners - 學習指南及課程摘要

本指南提供「AI Agents for Beginners」課程的摘要，並解釋構建 AI 代理的關鍵概念、框架及設計模式。

## 1. AI 代理簡介

**什麼是 AI 代理？**  
AI 代理是擴展大型語言模型（LLMs）能力的系統，賦予它們使用**工具**、**知識**與**記憶**的能力。與僅根據訓練數據生成文字的標準 LLM 聊天機器人不同，AI 代理能夠：  
- **感知**其環境（透過感測器或輸入）。  
- **推理**如何解決問題。  
- **行動**改變環境（透過執行器或工具執行）。  

**代理的關鍵組件：**  
- **環境**：代理運作的空間（例如：訂票系統）。  
- **感測器**：用於收集資訊的機制（例如：讀取 API）。  
- **執行器**：用於執行行動的機制（例如：發送電郵）。  
- **大腦（LLM）**：進行規劃和決策的推理引擎。  

## 2. 代理框架

本課程介紹了三個主要的代理構建框架：

| 框架 | 重點 | 適用範圍 |
|-----------|-------|----------|
| **Semantic Kernel** | 適用於生產環境的 .NET/Python SDK | 企業應用，將 AI 與現有代碼整合。 |
| **AutoGen** | 多代理協作 | 需要多個專家代理互相對話的複雜場景。 |
| **Azure AI Agent Service** | 托管雲服務 | 具備內建狀態管理的安全、可擴展部署。 |

## 3. 代理設計模式

設計模式有助於結構化代理的運作，可靠地解決問題。

### **工具使用模式**（第 4 課）  
此模式允許代理與外部世界互動。  
- **概念**：代理會獲得一個「結構」(schema)，列出可用函數及其參數。LLM 根據使用者請求決定*呼叫哪個工具*以及*使用何種參數*。  
- **流程**：使用者請求 -> LLM -> **工具選擇** -> **工具執行** -> LLM（帶工具輸出）-> 最終回應。  
- **使用案例**：取得即時資料（天氣、股價）、執行計算、運行程式碼。  

### **規劃模式**（第 7 課）  
此模式讓代理處理複雜多步驟的任務。  
- **概念**：代理將高階目標拆解成一連串更小的子任務。  
- **方法**：  
  - **任務拆解**：將「規劃行程」拆解為「訂機票」、「訂酒店」、「租車」。  
  - **迭代規劃**：根據先前步驟輸出重評規劃（例如機票已滿須改訂另一天）。  
- **實作**：通常涉及「規劃者」代理生成結構化計劃（例如 JSON），再由其他代理執行該計劃。  

## 4. 設計原則

設計代理時，請考慮三個維度：  
- **空間**：代理應連結人與知識，既可接近又不突兀。  
- **時間**：代理應從*過去*學習，提供當下（*現在*）的適切提醒，並為*未來*調整。  
- **核心**：接納不確定性，並透過透明和用戶控制建立信任。  

## 5. 重要課程重點總結

- **第 1 課**：代理是系統，不僅僅是模型。它們能感知、推理和行動。  
- **第 2 課**：像 Semantic Kernel 和 AutoGen 這類框架抽象化工具調用及狀態管理的複雜性。  
- **第 3 課**：設計時著重透明度及用戶控制。  
- **第 4 課**：工具是代理的「雙手」。結構定義對於 LLM 理解如何使用工具至關重要。  
- **第 7 課**：規劃是代理的「執行功能」，賦予其解決複雜工作流程的能力。
