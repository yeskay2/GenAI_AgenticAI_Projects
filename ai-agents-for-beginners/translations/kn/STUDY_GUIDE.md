# ಆರಂಭಿಕರಿಗೆ AI ಏಜೆಂಟುಗಳು - ಅಧ್ಯಯನ ಮಾರ್ಗದರ್ಶಿ ಮತ್ತು ಕೋರ್ಸ್ ಸಾರಾಂಶ

ಈ ಮಾರ್ಗದರ್ಶಿ "ಆಯೆಜಂಟ್‌ಗಳಿಗೆ AI ಏಜೆಂಟ್‌ಗಳು" ಕೋರ್ಸ್‌ನ ಸಾರಾಂಶವನ್ನು ನೀಡುತ್ತದೆ ಮತ್ತು AI ಏಜೆಂಟ್‌ಗಳನ್ನು ನಿರ್ಮಿಸಲು ಪ್ರಮುಖ ತತ್ವಗಳು, ಫ್ರೇಮ್ವರ್ಕ್‌ಗಳು ಮತ್ತು ವಿನ್ಯಾಸ ಮಾದರಿಗಳನ್ನು ವಿವರಿಸುತ್ತದೆ.

## 1. AI ಏಜೆಂಟ್‌ಗಳ ಪರಿಚಯ

**AI ಏಜೆಂಟ್‌ಗಳು ಏನು?**  
AI ಏಜೆಂಟ್‌ಗಳು ದೊಡ್ಡ ಭಾಷಾ ಮಾದರಿಗಳ (LLMs) ಸಾಮರ್ಥ್ಯಗಳನ್ನು ವಿಸ್ತರಿಸುವ ವ್ಯವಸ್ಥೆಗಳು, ಅವರಿಗೆ **ಉಪಕರಣಗಳು**, **ಜ್ಞಾನ**, ಮತ್ತು **ಸ್ಮೃತಿ** ಗಳಿಗೆ ಪ್ರಾಪ್ತಿಯನ್ನು ನೀಡುತ್ತವೆ. ತರಬೇತಿ ಡೇಟಾದ ಆಧಾರದ ಮೇಲೆ ಮಾತ್ರ ಪಠ್ಯವನ್ನು ರಚಿಸುವ ಸಾಮಾನ್ಯ LLM ಚಾಟ್‌ಬಾಟ್‌ನಂತೆ ಅಲ್ಲ, AI ಏಜೆಂಟ್ ಮಾಡಬಹುದು:  
- ತನ್ನ ಪರಿಸರವನ್ನು **ಗಮನಿಸುವುದು** (ಸೆನ್ಸರ್‌ಗಳು ಅಥವಾ ಇನ್‌ಪುಟ್‌ಗಳ ಮೂಲಕ).  
- ಸಮಸ್ಯೆಯನ್ನು ಹೇಗೆ ಬಗೆಹರಿಸಬೇಕೆಂದು **ತರ್ಕಿಸುವುದು**.  
- ಪರಿಸರವನ್ನು **ಬदಲಾಗಿಸಲು ಕ್ರಮ ಕೈಗೊಳ್ಳುವುದು** (ಆಕ್ಟ್ಯುವೇಟರ್‌ಗಳು ಅಥವಾ ಉಪಕರಣ ಕಾರ್ಯಗತಗೊಳಿಸುವಿಕೆಯಿಂದ).

**ಏಜೆಂಟ್‌ನ ಪ್ರಮುಖ ಘಟಕಗಳು:**  
- **ಪರಿಸರ**: ಏಜೆಂಟ್ ಕಾರ್ಯನಿರ್ವಹಿಸುವ ಸ್ಥಳ (ಉದಾ: ಬುಕ್ಕಿಂಗ್ ವ್ಯವಸ್ಥೆ).  
- **ಸೆನ್ಸರ್‌ಗಳು**: ಮಾಹಿತಿಯನ್ನು ಸಂಗ್ರಹಿಸುವ ಯಂತ್ರಗಳು (ಉದಾ: API ಓದುವಿಕೆ).  
- **ಆಕ್ಟ್ಯುವೇಟರ್‌ಗಳು**: ಕ್ರಮಗಳನ್ನು ಕೈಗೊಳ್ಳುವ ಯಂತ್ರಗಳು (ಉದಾ: ಇಮೇಲ್ ಕಳುಹಿಸುವುದು).  
- **ಮೂಳ (LLM)**: ಯೋಜನೆ ರೂಪಿಸುವ ಮತ್ತು ಯಾವ ಕ್ರಮ ಕೈಗೊಳ್ಳುವುದು ಎಂದು ನಿರ್ಧರಿಸುವ ತರ್ಕಯಂತ್ರ.

## 2. ಏಜೆಂಟಿಕ್ ಫ್ರೇಮ್ವರ್ಕ್‌ಗಳು

ಕೋರ್ಸ್ ಏಜೆಂಟ್‌ಗಳನ್ನು ನಿರ್ಮಿಸಲು ಮೂರು ಪ್ರಮುಖ ಫ್ರೇಮ್ವರ್ಕ್‌ಗಳನ್ನು ಒಳಗೊಂಡಿದೆ:

| ಫ್ರೇಮ್ವರ್ಕ್ | ಗಮನ ಕೇಂದ್ರ | ಉತ್ತಮ ಉತ್ಪತ್ತಿಗೆ |
|------------|-------------|------------------|
| **Semantic Kernel** | .NET/Python ಗೆ ಉತ್ಪಾದನಾ ಸಿದ್ಧ SDK | ಎಂಟರ್ಪ್ರೈಸ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳು, ಇತರ ಕೋಡ್‌ಗಳೊಂದಿಗೆ AI ಭೇಟಿಸಲು. |
| **AutoGen** | ಬಹು ಏಜೆಂಟ್ ಸಹಕಾರ | ಅನೇಕ ವಿಶೇಷ ಏಜೆಂಟ್‌ಗಳು ಪರಸ್ಪರ ಸಂವಾದ ಮಾಡುತ್ತಿರುವ ಕಠಿಣ ಪರಿಸ್ಥಿತಿಗಳು. |
| **Azure AI Agent Service** | ನಿರ್ವಹಿತ ಕ್ಲೌಡ್ ಸೇವೆ | ಸುರಕ್ಷಿತ, ವಿಸ್ತಾರಗೊಳ್ಳುವ ಪ್ರಕ್ರಿಯೆ ಮತ್ತು ಸ್ಥಿತಿ ನಿರ್ವಹಣೆಯೊಂದಿಗೆ. |

## 3. ಏಜೆಂಟಿಕ್ ವಿನ್ಯಾಸ ಮಾದರಿಗಳು

ವಿನ್ಯಾಸ ಮಾದರಿಗಳು ಏಜೆಂಟ್‌ಗಳು ಹೇಗೆ ಕಾರ್ಯನಿರ್ವಹಿಸಿ ಸಮಸ್ಯೆಗಳನ್ನು ನಿಭಾಯಿಸುತ್ತವೆ ಎಂಬುದನ್ನು ರಚಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತವೆ.

### **ಉಪಕರಣ ಉಪಯೋಗ ಮಾದರಿ** (ಪಾಠ 4)  
ಈ ಮಾದರಿ ಏಜೆಂಟ್‌ಗಳನ್ನು ಹೊರಗಿನ ಬಾಹ್ಯ ಲೋಕದೊಂದಿಗೆ ಪರಸ್ಪರ ಕ್ರಿಯೆ ನಡೆಸಲು ಸಾದ್ಯಮಾಡುತ್ತದೆ.  
- **ಧಾರಣೆ**: ಏಜೆಂಟ್‌ಗೆ “ವ್ಯವಸ್ಥೆ” (ಲಭ್ಯವಿರುವ ಕಾರ್ಯಕ್ರಮಗಳು ಮತ್ತು ಅವುಗಳ ಪ್ಯಾರಾಮೀಟರ್‌ಗಳ ಪಟ್ಟಿ) ನೀಡಲಾಗುತ್ತದೆ. LLM ಯಾವ ಉಪಕರಣವನ್ನು ಕರೆಸುವುದು ಮತ್ತು ಯಾವುದಾದರಿಂದ ಉಪಕರಣವನ್ನು ಬಳಸುವುದು ಎಂದು ನಿರ್ಧರಿಸುತ್ತದೆ.  
- **ಪ್ರಕ್ರಿಯೆ**: ಬಳಕೆದಾರ ಮನವಿ -> LLM -> **ಉಪಕರಣ ಆಯ್ಕೆ** -> **ಉಪಕರಣ ನಡತೆ** -> LLM (ಉಪಕರಣದ ಔಟ್‌ಪುಟ್‌ ಸಹಿತ) -> ಅಂತಿಮ ಉತ್ತರ.  
- **ಬಳಕೆ ಪ್ರಕರಣಗಳು**: ನೈಜ-ಕಾಲದಲ್ಲಿ ಡೇಟಾ (ಹವಾಮಾನ, ಷೇರು ಬೆಲೆ), ಲೆಕ್ಕಾಚಾರ, ಕೋಡ್ ಕಾರ್ಯಗತಗೊಳಿಸುವಿಕೆ.

### **ಯೋಜನೆ ಮಾದರಿ** (ಪಾಠ 7)  
ಈ ಮಾದರಿ ಏಜೆಂಟ್‌ಗಳಿಗೆ ಸಂಕೀರ್ಣ, ಬಹು ಹಂತದ ಕಾರ್ಯಗಳನ್ನು ನಿಭಾಯಿಸಲು ಸಾಧ್ಯಮಾಡುತ್ತದೆ.  
- **ಧಾರಣೆ**: ಏಜೆಂಟ್ ಉನ್ನತ ಮಟ್ಟದ ಗುರಿಯನ್ನು ಸಣ್ಣ ಉಪಕಾರ್ಯಗಳ ಸರಣಿಯಾಗಿ ವಿಭಜಿಸುತ್ತದೆ.  
- **ಪ್ರವೃತ್ತಿಗಳು**:  
  - **ಕಾರ್ಯ ವಿಭಜನೆ**: “ಪ್ರಯಾಣ ಯೋಜನೆ” ಅನ್ನು “ಬೆಲೆ ಬುಕ್ ಮಾಡಿ”, “ಹೋಟೆಲ್ ಬುಕ್ ಮಾಡಿ”, “ಕಾರ್ವಾಡಿಕೆ”ಗೆ ಬಿಸಿಯಾಗಿ ವಿಭಜಿಸುವುದು.  
  - **ಪರ್ಯಾಯ ಯೋಜನೆ**: ಹಿಂದಿನ ಹಂತಗಳ ಔಟ್‌ಪುಟ್ ಆಧರಿಸಿ ಯೋಜನೆಯನ್ನು ಮರುಮೂಲೆಮಾಡುವುದು (ಹುಡುಕಾಟದ ದಿನಾಂಕ ತುಂಬಿದ್ದರೆ ಬೇರೊಂದು ದಿನಾಂಕ ಆರಿಸುವುದು).  
- **ಕಾರ್ಯಗತಗೊಳಿಸುವಿಕೆ**: ಸಾಮಾನ್ಯವಾಗಿ "ಯೋಜಕ" ಎಂಬ ಏಜೆಂಟ್ ಒಬ್ಬನು ರಚಿತ ಯೋಜನೆ (ಉದಾ: JSON) ರಚಿಸಿ, ಅದನ್ನು ಇತರ ಏಜೆಂಟ್‌ಗಳು ಕಾರ್ಯಗತಗೊಳಿಸುತ್ತವೆ.

## 4. ವಿನ್ಯಾಸ ತತ್ವಗಳು

ಏಜೆಂಟ್‌ಗಳನ್ನು ವಿನ್ಯಾಸ ಮಾಡುವಾಗ ಮೂರು ಆಯಾಮಗಳನ್ನು ಪರಿಗಣಿಸಬೇಕು:  
- **ಸ್ಥಳ**: ಏಜೆಂಟ್‌ಗಳು ಜನರನ್ನು ಹಾಗೂ ಜ್ಞಾನವನ್ನು ಸಂಪರ್ಕಿಸಬೇಕು, ಸುಲಭವಾಗಿ ಲಭ್ಯವಿರಬೇಕು ಆದರೆ ಅತಿಯಾದ ಹಸ್ತಕ್ಷೇಪವಿಲ್ಲದೆ ಇರಬೇಕು.  
- **ಸಮಯ**: ಏಜೆಂಟ್‌ಗಳು **ಹಿಂದಿನ** ಅನುಭವದಿಂದ ಕಲಿಯಬೇಕು, **ಇದೀಗ** ಸೂಕ್ತ ಸೂಚನೆಗಳು ನೀಡಬೇಕು, ಮತ್ತು **ಭವಿಷ್ಯಕ್ಕೆ** ಹೊಂದಿಕೊಳ್ಳಬೇಕು.  
- **ಹೃದಯ**: ಅನುಮಾನಗಳನ್ನು ಪರಿಗಣಿಸಿ, ಆದರೆ ಬರುವಿಕೆಯನ್ನು திறಮಯತೆ ಮತ್ತು ಬಳಕೆದಾರ ನಿಯಂತ್ರಣದ ಮೂಲಕ ಸ್ಥಾಪಿಸಬೇಕು.

## 5. ಪ್ರಮುಖ ಪಾಠಗಳ ಸಾರಾಂಶ

- **ಪಾಠ 1**: ಏಜೆಂಟ್‌ಗಳು ವ್ಯವಸ್ಥೆಗಳೇ, ಮಾದರಿಗಳು ಮಾತ್ರವಲ್ಲ. ಅವು ಗಮನಿಸಬಹುದಾದವು, ಕಾರಣ ಮಾಡಬಹುದಾದವು, ಮತ್ತು ಕ್ರಮ ಕೈಗೊಳ್ಳಬಹುದಾದವು.  
- **ಪಾಠ 2**: Semantic Kernel ಮತ್ತು AutoGen ಹ್ಮೀಸಲು ಉಪಕರಣ ಕರೆಗು ಮತ್ತು ಸ್ಥಿತಿ ನಿರ್ವಹಣೆಯ ಸಂಕೀರ್ಣತೆಯನ್ನು ಸರಳಗೊಳಿಸುತ್ತವೆ.  
- **ಪಾಠ 3**: ಪಾರದರ್ಶಕತೆ ಮತ್ತು ಬಳಕೆದಾರ ನಿಯಂತ್ರಣವನ್ನು ಮನಸ್ಸಿನಲ್ಲಿ ಇಟ್ಟು ವಿನ್ಯಾಸ ಮಾಡಿ.  
- **ಪಾಠ 4**: ಉಪಕರಣಗಳು ಏಜೆಂಟ್‌ನ "ಕೈಗಳು". LLM ಅವುಗಳನ್ನು ಹೇಗೆ ಬಳಸಬೇಕೆಂದು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು "ವ್ಯವಸ್ಥೆ" ವ್ಯಾಖ್ಯಾನ ಅತ್ಯಂತ ಮಹತ್ವದ್ದು.  
- **ಪಾಠ 7**: ಯೋಜನೆ ಏಜೆಂಟ್‌ನ "ನಿರ್ವಹಣಾ ಕಾರ್ಯ" ಆಗಿದ್ದು ಸಂಕೀರ್ಣ ಕೆಲಸಗಳನ್ನು ನಿಭಾಯಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.
