# AI 代理入門 - 學習指南與課程摘要

本指南提供「AI 代理入門」課程摘要，並說明構建 AI 代理的關鍵概念、框架與設計模式。

## 1. AI 代理簡介

**什麼是 AI 代理？**  
AI 代理是透過賦予大型語言模型（LLM）**工具**、**知識**和**記憶**的能力，來擴充其功能的系統。與僅根據訓練資料生成文本的標準 LLM 聊天機器人不同，AI 代理能夠：  
- **感知**環境（透過感測器或輸入）。  
- **推理**如何解決問題。  
- **行動**以改變環境（透過驅動器或工具執行）。  

**代理的關鍵組成部分：**  
- **環境**：代理運作的空間（例如訂房系統）。  
- **感測器**：用於蒐集資訊的機制（例如讀取 API）。  
- **驅動器**：執行動作的機制（例如發送電子郵件）。  
- **大腦（LLM）**：規劃與決策採取哪些行動的推理引擎。  

## 2. 代理框架

本課程介紹三個主要的代理建構框架：

| 框架 | 重點 | 適用範圍 |
|-----------|-------|----------|
| **Semantic Kernel** | 適用於 .NET/Python 的生產就緒 SDK | 企業應用，將 AI 整合至現有程式碼中。 |
| **AutoGen** | 多代理協作 | 需多種專門代理相互溝通的複雜場景。 |
| **Azure AI Agent Service** | 管理雲端服務 | 安全且具可擴展性的部署，內建狀態管理。 |

## 3. 代理設計模式

設計模式有助於結構化代理如何運作，以可靠地解決問題。

### **工具使用模式** (第4課)  
此模式使代理能夠與外部世界互動。  
- **概念**：代理收到「結構描述」（包含可用函式及其參數列表）。LLM 根據用戶請求決定*使用哪個工具*及*使用何種參數*。  
- **流程**：用戶請求 -> LLM -> **工具選擇** -> **工具執行** -> LLM（帶工具輸出）-> 最終回覆。  
- **使用案例**：擷取即時資料（天氣、股價）、執行計算、執行程式碼。  

### **規劃模式** (第7課)  
此模式使代理能解決複雜的多步驟任務。  
- **概念**：代理將一個高階目標拆解成一連串較小的子任務。  
- **方法**：  
  - **任務分解**：將「規劃旅行」拆分為「訂機票」、「訂飯店」、「租車」。  
  - **反覆規劃**：根據前一步結果重新評估計畫（例如若機票額滿，改訂其他日期）。  
- **實作**：通常涉及一個「規劃者」代理產生結構化計畫（例如 JSON），由其他代理執行。  

## 4. 設計原則

設計代理時，需考慮三個維度：  
- **空間**：代理應該串連人與知識，易於接觸且不造成干擾。  
- **時間**：代理應從*過去*學習，在*當下*提供相關提示，並為*未來*做出調整。  
- **核心**：接受不確定性，但透過透明度及用戶控制建立信任。  

## 5. 重要課程總結

- **第1課**：代理是系統，不僅是模型。它們能感知、推理與行動。  
- **第2課**：Semantic Kernel 與 AutoGen 等框架抽象了工具調用與狀態管理的複雜性。  
- **第3課**：設計時須考慮透明度與用戶控制。  
- **第4課**：工具是代理的「手」，結構描述定義對於 LLM 正確使用它們非常關鍵。  
- **第7課**：規劃是代理的「執行功能」，讓代理能處理複雜工作流程。
